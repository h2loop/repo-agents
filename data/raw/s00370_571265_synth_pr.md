## Title: Add bounds checking to EMMAS-SAP primitive string conversion

### Summary
The `emmas_to_str()` function in the NAS UE EMM SAP layer performs unchecked array indexing when converting EMMAS primitive values to human-readable strings. When invalid primitive values are passed due to NGAP interface errors or corrupted messages, this results in out-of-bounds memory access, causing undefined behavior and potential NGAP procedure failures downstream.

This fix adds proper bounds validation before array access. If the primitive value is outside the valid range [0, 14], the function now logs a warning and returns "EMMAS_UNKNOWN" instead of accessing invalid memory. This prevents memory corruption and provides better diagnostics for NGAP interface issues.

### Changes
- `openair3/NAS/UE/EMM/SAP/emm_as.c`: Added bounds check in `emmas_to_str()` to validate `emmas` parameter against array size before indexing. Returns "EMMAS_UNKNOWN" and logs warning on invalid values.

### Implementation Details
- Uses `sizeof()` operator to automatically determine array bounds, making the check robust to future array modifications
- Logs invalid primitive values via `LOG_TRACE(WARNING, ...)` for debugging NGAP interface issues
- Minimal performance impact: adds single integer comparison to fast path

### Testing
- Static syntax analysis confirms correct C language usage
- Code review validates bounds logic covers negative indices and overflow cases
- Maintains full backward compatibility for all valid EMMAS primitive values
