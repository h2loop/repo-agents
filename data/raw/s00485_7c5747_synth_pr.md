## Title: Fix invalid PLMN initialization causing NAS protocol state machine violations

### Summary
The `make_plmn_from_conf()` function initializes PLMN structures with `memset(&plmn, 0xff, sizeof(plmn))`, which fills all digits with invalid BCD values. According to 3GPP TS 24.008, PLMN digits must be BCD-encoded (0-9), and 0xFF is not a valid digit. When these malformed PLMNs are used in NAS protocol state machines, they cause specification violations and unpredictable state transitions. Additionally, for 2-digit MNCs, the third MNC digit remained as 0xFF instead of the proper filler value.

This fix ensures 3GPP compliance by initializing the PLMN structure to zero and explicitly setting the third MNC digit to 'F' (0xF in BCD, representing an unused digit) for 2-digit MNC configurations. This prevents invalid PLMN values from propagating to NAS state machines.

### Changes
- `openair3/NAS/TOOLS/conf_network.c`: 
  - Changed `memset(&plmn, 0xff, sizeof(plmn))` to `memset(&plmn, 0, sizeof(plmn))` for proper zero initialization
  - Added `else { plmn.MNCdigit3 = 'F'; }` clause to set the third MNC digit to the standard BCD filler value when MNC length is 2 digits

### Testing
- This is a protocol compliance fix ensuring valid BCD encoding per 3GPP specifications
- Code review confirms PLMN digits now contain valid values (0-9 or 0xF filler) instead of invalid 0xFF values
- No functional behavior change for valid PLMN configurations; only eliminates specification violations for malformed cases
