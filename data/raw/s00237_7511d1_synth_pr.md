## Title: Replace magic number with symbolic constant for LTE channel estimation symbols

### Summary
Replace the hardcoded magic number `3` (representing the number of OFDM symbols used for LTE channel estimation) with a descriptive symbolic constant `LTE_NUM_ESTIMATION_SYMBOLS` across the UE measurements module. This magic number appeared in multiple loop bounds throughout the codebase, making the intent unclear and creating maintenance risk if this value ever needs to change. Using a named constant improves code readability and ensures future modifications only need to be made in one place.

### Changes
- `openair1/PHY/LTE_ESTIMATION/lte_ue_measurements.c`: 
  - Added `#define LTE_NUM_ESTIMATION_SYMBOLS 3` with explanatory comment at the top of the file
  - Replaced all instances of the magic number `3` in channel estimation loops with the new symbolic constant in these functions:
    - `conjch0_mult_ch1()` (line 471)
    - `construct_HhH_elements()` (line 517)
    - `squared_matrix_element()` (line 566)
    - `det_HhH()` (line 603)
    - `numer()` (line 648)
    - `dlsch_channel_level_TM34_meas()` (line 702)

### Implementation Details
The constant is defined locally in the C file rather than in a header since it's specific to LTE channel estimation implementation details within this module. All replacements maintain identical functionality—the change is purely semantic with no behavioral impact.

- Verified successful compilation of the modified file
- Confirmed all instances of the magic number in estimation symbol loops have been replaced
