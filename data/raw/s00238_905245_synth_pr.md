## Title: Fix signed/unsigned integer comparison mismatch in CSI resource config lookup

### Summary
The `find_CSI_resourceconfig()` function in the UE MAC scheduler used an unsigned return type (`NR_CSI_ResourceConfigId_t`) but returned -1 as a sentinel value when no matching configuration was found. This created a signed/unsigned comparison mismatch at the call site where the result was checked with `if(csi_res_id < 0)`. With an unsigned type, this comparison would always evaluate to false, causing incorrect branch behavior and potentially scheduling CSI-RS reception for invalid configurations.

This fix changes the return type to signed `int` to properly support the error signaling pattern. The function signature and the local variable receiving its return value are both updated to maintain type consistency.
