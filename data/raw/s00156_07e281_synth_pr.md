## Title: Fix file descriptor leak in itti_create_task error paths

### Summary
Replace the `threadCreate` wrapper in `itti_create_task` with explicit pthread creation to fix resource leaks on error paths. The original implementation used `AssertFatal` for error handling, which aborted the process without cleaning up pthread attributes or file descriptors. The new code adds proper error checking and cleanup, returning -1 on failure instead of crashing.

### Changes
- `common/utils/ocp_itti/intertask_interface.cpp`: 
  - Replaced `threadCreate()` call with manual pthread attribute initialization and thread creation
  - Added error handling for `pthread_attr_init`, `pthread_attr_setinheritsched`, `pthread_attr_setschedpolicy`, `pthread_attr_setschedparam`, and `pthread_create`
  - Added `pthread_attr_destroy()` calls on all error paths to prevent attribute memory leaks
  - Added descriptive error logging via `LOG_E()` instead of fatal assertions
  - Maintains identical thread behavior (SCHED_RR policy, real-time priority, thread naming)

### Implementation Details
The `threadCreate` helper encapsulated pthread setup but used `AssertFatal` macros that prevented proper cleanup on failure. The new inline implementation:
- Initializes pthread attributes and checks each configuration step
- On any failure, destroys attributes before returning -1
- Successfully created threads are still detached with real-time scheduling
- No functional change to the happy path; only error handling improved

### Testing
- Verified thread creation succeeds for all ITTI tasks during gNB startup
- Error injection tests confirm cleanup paths execute without memory leaks
- Thread names and scheduling parameters verified via `/proc/[pid]/task/[tid]/status`
