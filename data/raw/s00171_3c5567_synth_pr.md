## Title: Add NULL checks to prevent crashes in MMSE processing core

### Summary
The `mmse_processing_core` function in the LTE UE PHY layer performs multiple dynamic memory allocations without checking for allocation failures. When memory is exhausted, `malloc()` returns NULL, leading to null pointer dereferences that crash the UE. This fix adds comprehensive NULL checks after each allocation and implements proper cleanup to prevent memory leaks when failures occur.

### Changes
- `openair1/PHY/LTE_UE_TRANSPORT/dlsch_demodulation.c`: Added NULL validation for all `malloc()` calls in `mmse_processing_core()`. When any allocation fails, the function now frees previously allocated memory blocks in reverse order before returning early. Also added initial NULL checks for input pointer parameters `rxdataF_ext` and `dl_ch_estimates_ext`.

### Implementation Details
The fix follows a structured cleanup pattern: each allocation failure triggers deallocation of all previously allocated resources. For array allocations, partial cleanup handles elements allocated before the failure point. This ensures no memory leaks occur during error paths while preventing undefined behavior from NULL pointer dereferences.

### Testing
This defensive programming fix prevents crashes in low-memory conditions. The changes maintain existing functionality while adding robustness against allocation failures that may occur under system memory pressure or resource-constrained environments.
