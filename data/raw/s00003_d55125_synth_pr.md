## Title: Fix unchecked return values in S1AP decoder encoding functions

### Summary
The S1AP eNB decoder module contained multiple instances where the return value from `asn_encode_to_new_buffer()` was not checked before accessing the result buffer. This function can fail during ASN.1 encoding (e.g., due to memory allocation failures or invalid PDU structures), and the unchecked access could lead to segmentation faults or undefined behavior when `res.buffer` is NULL or contains invalid data.

The root cause was a pattern across several decoder functions where the code immediately called `free(res.buffer)` without verifying that encoding succeeded. This fix adds proper validation by checking `res.result.encoded > 0` before freeing the buffer and returns an error code with appropriate logging when encoding fails.

### Changes
- `openair3/S1AP/s1ap_eNB_decoder.c`: Added error handling for `asn_encode_to_new_buffer()` calls in:
  - `s1ap_eNB_decode_initiating_message()` (affects Downlink NAS Transport, Initial Context Setup, UE Context Release, Paging, and E-RAB procedures)
  - `s1ap_eNB_decode_outcome()` (affects various outcome messages)
  - `s1ap_eNB_decode_unsuccessful_outcome()` (affects unsuccessful outcome messages)
  
  Each call site now checks the encoded result before freeing the buffer and returns `-1` with `S1AP_ERROR` logging on failure.

### Implementation Details
The fix follows the established pattern: check if `res.result.encoded > 0` to confirm successful encoding before calling `free(res.buffer)`. On failure, log an error message with the procedure name and return `-1` to propagate the error upstream. This ensures consistent error handling across all S1AP message decoding paths.
