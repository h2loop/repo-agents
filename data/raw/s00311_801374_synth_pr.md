## Title: Fix textarea widget layout protocol implementation

### Summary
The textarea widget in the T tracer GUI subsystem was missing proper implementation of the layout management protocol. While it provided size hints through the `hints()` callback, it lacked an `allocate()` callback to receive its final allocated position and dimensions from the layout manager. Additionally, the widget structure did not store its actual allocated width and height, only storing desired dimensions and text metrics.

This patch completes the layout protocol implementation by:
- Adding `width` and `height` fields to `struct textarea_widget` to track allocated dimensions
- Implementing the missing `allocate()` callback that updates the widget's position and size
- Updating `textarea_set_text()` to properly capture text height metrics
- Initializing the new fields in `new_textarea()`

### Changes
- `common/utils/T/tracer/gui/gui_defs.h`: Added `width` and `height` fields to `struct textarea_widget`
- `common/utils/T/tracer/gui/textarea.c`: 
  - Implemented `allocate()` callback for layout manager integration
  - Updated `new_textarea()` to initialize new dimension fields
  - Modified `textarea_set_text()` to capture height from text metrics

### Testing
- Verified textarea widgets now properly receive and store allocated dimensions from the layout manager
- Confirmed text rendering still works correctly with the new dimension tracking
- Tested GUI layout behavior remains consistent with existing widget implementations
