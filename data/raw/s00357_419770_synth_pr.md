## Title: Fix type confusion in RLC UM callback function signatures

### Summary
The RLC UM reassembly logic uses callback functions to filter PDU segments based on sequence number criteria. The helper functions `less_than_vr_ur`, `outside_of_reordering_window`, and `less_than_vr_uh` were defined with concrete `rlc_entity_um_t *` parameters, but were being passed to `rlc_um_reassemble()` which expected a callback signature with `void *` entity parameter. This type mismatch created a subtle type confusion issue where incompatible function pointer types were being used, potentially leading to undefined behavior on platforms with strict pointer type checking or non-universal pointer representations.

This fix standardizes all callback functions to use the `void *` entity parameter pattern, matching the existing `sn_in_recv_window` implementation. Each callback now performs an explicit cast to `rlc_entity_um_t *` internally, ensuring type safety while maintaining the flexibility of the callback mechanism.

### Changes
- `openair2/LAYER2/rlc_v2/rlc_entity_um.c`: 
  - Updated function signatures of `less_than_vr_ur`, `outside_of_reordering_window`, and `less_than_vr_uh` to accept `void *_entity` parameter
  - Added explicit `(rlc_entity_um_t *)` cast inside each function before dereferencing
  - Modified `rlc_um_reassemble()` to accept `int (*check_sn)(void *entity, int sn)` callback signature

### Implementation Details
The fix follows the established pattern already used by `sn_in_recv_window()` in the same file. While the previous code likely worked on most platforms due to compatible pointer representations, this change eliminates the type confusion at the compiler level and improves code maintainability by enforcing consistent callback signatures throughout the RLC UM module.
