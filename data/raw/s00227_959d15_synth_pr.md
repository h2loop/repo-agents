## Title: Add critical failure logging to MCE ENB Configuration Update handler

### Summary
The `MCE_send_ENB_CONFIGURATION_UPDATE_FAILURE` function in the M2AP MCE interface management module is a stub that triggers `AssertFatal` when called, indicating an unimplemented critical failure path. However, it lacked any logging statement to help operators and developers diagnose when this failure condition occurs and what context led to it. This made debugging M2AP ENB configuration update failures impossible in production environments.

This patch adds an error-level log statement that records when the critical failure event is triggered, providing crucial visibility into M2AP protocol failures before the assertion terminates the process.

### Changes
- `openair2/M2AP/m2ap_MCE_interface_management.c`: Added `LOG_E(M2AP, "MCE_send_ENB_CONFIGURATION_UPDATE_FAILURE: Critical failure event\n");` at the entry point of `MCE_send_ENB_CONFIGURATION_UPDATE_FAILURE` to log critical failure events before the fatal assertion.

### Implementation Details
The log statement uses the standard `LOG_E` macro with the `M2AP` module identifier, consistent with error logging patterns throughout the file. The message clearly identifies the function name and marks it as a critical failure event, making it easily searchable in log aggregates.

### Testing
- Verified compilation succeeds with the new logging statement
- Confirmed log message format matches existing M2AP error logging patterns in the codebase
- The log will be emitted to the configured error logging destination before the assertion terminates execution
