## Title: Remove AssertFatal from unimplemented E1AP notification function

### Summary
The stub function `e1apCUUP_send_BEARER_CONTEXT_INACTIVITY_NOTIFICATION` unconditionally calls `AssertFatal`, which terminates execution and renders all downstream code paths unreachable. This prevents proper error handling in calling code and makes the function unsafe to call even in non-critical paths. Since this is an unimplemented feature, it should return an error code to allow graceful degradation rather than crashing the process.

### Changes
- `openair2/E1AP/e1ap.c`: Removed the `AssertFatal(false, "Not implemented yet\n");` statement from `e1apCUUP_send_BEARER_CONTEXT_INACTIVITY_NOTIFICATION()`. The function now simply returns `-1` to indicate the operation is not supported, enabling callers to handle the error appropriately.

### Testing
- Verified that the modified function compiles without warnings
- Confirmed that call sites can now handle the returned error code instead of encountering process termination
- No functional impact expected as this feature was previously non-operational due to the fatal assertion
