## Title: Add error propagation and validation in E2 node type detection

### Summary
The `get_e2_node_type()` function in the O-RAN RAN Function module previously lacked proper error handling, potentially leading to silent failures from NULL pointer dereferences or invalid node type values. This change adds comprehensive validation and error propagation to catch configuration issues early and prevent undefined behavior.

The function determines the E2 node type based on compile-time flags and runtime RRC state, but could silently return invalid values or dereference NULL pointers. Downstream code at the call site in `read_rc_setup_sm()` also failed to validate the function pointer before invocation.

### Changes
- `openair2/E2AP/RAN_FUNCTION/O-RAN/ran_func_rc.c`:
  - In `get_e2_node_type()`: Added NULL check for `RC.nrrrc[0]` before dereferencing, with error message and assertion
  - Added fallback error handling when no NGRAN compile flags are defined
  - Added range validation for the returned `node_type` value against `END_NGRAN_NODE_TYPE`
  - In `read_rc_setup_sm()`: Added validation that `ran_def_rc[node_type]` function pointer is non-NULL before calling it

### Implementation Details
All error conditions now print descriptive messages to stderr and trigger assertions to fail fast during development/testing, rather than causing subtle failures or crashes later. The validation ensures:
1. RRC instance exists before accessing its `node_type` field
2. The resolved node type is within the valid enum range
3. The RAN function definition pointer for that node type is properly initialized

This replaces silent ignored failures with explicit, debuggable error conditions.
