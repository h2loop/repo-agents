## Title: Fix NULL pointer dereference before allocation check in rr_dl_setup

### Summary
Reorder the NULL check and pointer dereference in `rr_dl_setup()` to prevent segmentation faults and restore proper error handling. The original code dereferenced the malloc result before validating it, causing two critical issues: (1) if allocation fails, the code crashes on `*(int *) data = 0` before reaching the AssertFatal, and (2) the AssertFatal becomes dead code that never catches allocation failures. This fix ensures proper failure detection and prevents undefined behavior during memory exhaustion scenarios.

### Changes
- `openair2/LAYER2/MAC/pre_processor.c`: In `rr_dl_setup()`, moved `AssertFatal(data, ...)` immediately after `malloc()` and before the pointer dereference `*(int *) data = 0`. This restores the intended error handling flow where allocation failures are caught and reported rather than causing silent crashes.

### Implementation Details
The fix corrects the operation sequence from:
1. Allocate memory
2. Dereference pointer (unsafe)
3. Check for NULL (dead code)

To the correct pattern:
1. Allocate memory
2. Check for NULL
3. Dereference pointer (safe)

This follows standard defensive programming practices and matches the pattern used elsewhere in the codebase. The function now properly initializes the allocated integer to zero only after confirming allocation success.

### Testing
- Verified the change compiles without warnings
- Code inspection confirms the NULL check now precedes all pointer dereferences
- This pattern matches other setup functions in the MAC layer for consistency
