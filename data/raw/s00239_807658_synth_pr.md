## Title: Fix buffer overflow vulnerability in NR small block encoder

### Summary
The `encodeSmallBlock` function in the NR small block coding module contains a buffer overflow vulnerability due to unchecked array indexing. The function iterates up to `len` times and accesses `nrSmallBlockBasis[i]` without bounds validation. Since the `nrSmallBlockBasis` array contains only 11 elements (indices 0-10), passing a `len` parameter greater than 11 causes an out-of-bounds read, leading to undefined behavior and potential security issues.

This fix adds proper bounds checking by calculating the maximum valid index based on the array size and modifying the loop condition to ensure we never read beyond the array bounds. The change maintains the original encoding logic while preventing memory safety violations.

### Changes
- `openair1/PHY/CODING/nrSmallBlock/encodeSmallBlock.c`: Added bounds checking in `encodeSmallBlock()` by introducing `max_len` constant and updating the loop condition to `i < len && i < max_len`, preventing out-of-bounds access to `nrSmallBlockBasis`.

### Testing
- Verified the fix compiles successfully within the OAI build system
- The bounds check ensures the function cannot read beyond the 11-element `nrSmallBlockBasis` array regardless of the `len` parameter value
- No functional regression expected as valid calls with `len <= 11` behave identically to the original implementation
