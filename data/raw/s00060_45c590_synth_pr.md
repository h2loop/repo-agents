## Title: Fix missing null terminator in debug filename buffer

### Summary
Fix a potential buffer overflow and out-of-bounds read in the debug code path of `nr_rx_pdsch`. The `filename` buffer used for debug output files was sized at 50 bytes, which could be insufficient for the formatted strings, and lacked explicit null termination after `snprintf` calls. This could lead to out-of-bounds reads when the filename string is consumed by `write_output()` and other file operations.

The fix increases the buffer size to 100 bytes, uses `sizeof(filename)` for bounds checking, and adds explicit null termination after each `snprintf` to ensure string safety regardless of truncation behavior.

### Changes
- `openair1/PHY/NR_UE_TRANSPORT/nr_dlsch_demodulation.c`: 
  - Increased `filename` buffer size from 50 to 100 bytes in the `DEBUG_PDSCH_RX` section
  - Updated all 6 `snprintf()` calls to use `sizeof(filename)` instead of hardcoded size
  - Added explicit null terminator assignment after each `snprintf()` for defensive programming

### Implementation Details
The changes are isolated to the `DEBUG_PDSCH_RX` preprocessor block and do not affect normal operation. The explicit null termination (`filename[sizeof(filename)-1] = '\0'`) guards against any edge cases where `snprintf` might not null-terminate (e.g., when the formatted string exactly fills the buffer). Using `sizeof()` ensures the buffer size is always correct if the buffer size changes again in the future.

### Testing
This fix only affects debug builds with `DEBUG_PDSCH_RX` defined. The code compiles successfully and the changes are verified to be isolated to debug output paths.
