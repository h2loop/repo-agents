## Title: Replace magic number with symbolic constant in MCE_APP itti message allocation

### Summary
Replace the magic number `0` with the symbolic constant `INSTANCE_DEFAULT` for instance identifiers in MCE_APP module itti message allocations. This improves code readability and maintainability by making the purpose of the parameter explicit.

The second parameter to `itti_alloc_new_message()` represents the instance ID for the message. Using `0` directly as a magic number provides no semantic meaning to readers, whereas `INSTANCE_DEFAULT` clearly indicates the default instance context. This change aligns with the existing codebase patterns where symbolic constants are preferred over raw numeric values.

### Changes
- `openair2/MCE_APP/mce_app.c`: Replaced all occurrences of `0` with `INSTANCE_DEFAULT` in calls to `itti_alloc_new_message()` across multiple functions including `MCE_app_register()`, `MCE_app_register_m3()`, `MCE_app_handle_m3ap_mbms_session_stop_req()`, `MCE_app_handle_m3ap_mbms_session_update_req()`, and `MCE_app_send_m2ap_session_start_req()`.

### Implementation Details
The `INSTANCE_DEFAULT` constant is defined in the intertask interface headers (`intertask_interface.h`) and represents the default instance identifier (value 0). This change is purely cosmetic and does not alter any functional behavior, as the numeric value remains the same. All modified code paths maintain identical execution semantics while gaining improved self-documentation.
