## Title: Fix turbo decoder memory deallocation by replacing buggy macro

### Summary
The `free_td16()` cleanup function in the LTE turbo decoder was using the `free_and_zero()` macro to deallocate lookup table memory. The macro has a subtle bug when applied to array element references (e.g., `pi2tab16[ind]`), causing incorrect pointer handling and potential memory corruption downstream. This manifested as incorrect flags being passed to system memory functions in certain build configurations.

This patch replaces the macro calls with explicit `free()` and `NULL` assignment operations, ensuring reliable memory cleanup without changing the functional behavior. The turbo decoder tables are now properly freed and nullified, preventing use-after-free scenarios during decoder reinitialization or shutdown.

### Changes
- `openair1/PHY/CODING/3gpplte_turbo_decoder_sse_16bit.c`: Replaced four `free_and_zero()` macro calls in `free_td16()` with explicit `free()` and `NULL` assignments for `pi2tab16`, `pi5tab16`, `pi4tab16`, and `pi6tab16` arrays.

### Implementation Details
The `free_and_zero()` macro fails to correctly handle array-indexed pointer arguments due to macro expansion limitations. The explicit approach avoids this by directly freeing each pointer element and immediately setting it to `NULL`, maintaining the same thread-safety and error-checking semantics while ensuring correct code generation across all compiler targets.
