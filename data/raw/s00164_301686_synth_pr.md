## Title: Clear PLMN index after PLMN selection procedure completion

### Summary
The PLMN selection procedure in the NAS UE stack failed to clear the `index` field in the PLMN list structure after successful completion. This caused stale state to persist, which could lead to incorrect behavior during subsequent PLMN selection attempts. The `index` tracks the current scanning position during PLMN searches and must be reset when the procedure completes to ensure clean state for future operations.

This fix clears the PLMN list index immediately after the SAP message is sent to indicate procedure completion, preventing stale state from affecting subsequent PLMN selection procedures.

### Changes
- `openair3/NAS/UE/EMM/IdleMode.c`: Added `emm_plmn_list->index = 0;` in `emm_proc_plmn_selection_end()` after calling `emm_sap_send()` to reset the PLMN list index when the procedure completes.

### Implementation Details
The index reset is placed immediately after `emm_sap_send()` to ensure the state is cleared at the logical completion point of the PLMN selection procedure, before any subsequent PLMN list operations can access the stale index value. This maintains proper state hygiene without affecting the ongoing SAP message processing.

### Testing
- Verified that the PLMN index is properly reset after successful PLMN selection
- Confirmed no regression in PLMN selection and reselection procedures
- Tested with multiple consecutive PLMN selection attempts to ensure clean state between procedures
