## Title: Fix signedness bug in NFAPI shared PA parameter serialization

### Summary
The NFAPI PNF parameter general TLV incorrectly serializes the `shared_pa` field using unsigned 8-bit packing functions (`push8`/`pull8`). This field represents a power adjustment value that can be negative, and using unsigned serialization corrupts negative values during message packing/unpacking. The bug manifests when the PHY attempts to apply power control adjustments, resulting in incorrect transmit power levels being configured downstream.

The fix changes the serialization to use signed 8-bit functions (`pushs8`/`pulls8`) to properly handle the signed nature of power adjustment values.

### Changes
- `nfapi/open-nFAPI/common/src/nfapi.c`: 
  - Line 1441: Changed `push8(value->shared_pa, ...)` to `pushs8(value->shared_pa, ...)` in `pack_pnf_param_general_value()`
  - Line 1461: Changed `pull8(ppReadPackedMsg, &value->shared_pa, ...)` to `pulls8(ppReadPackedMsg, &value->shared_pa, ...)` in `unpack_pnf_param_general_value()`

### Implementation Details
The `shared_pa` field is defined as a signed integer in the NFAPI specification to allow for both positive and negative power adjustments. The unsigned packing functions were treating negative values as large positive numbers (due to two's complement representation), causing the PHY to apply incorrect power offsets. The signed variants correctly preserve the intended power adjustment values during serialization.

### Testing
This is a type correctness fix that ensures proper handling of signed power values. The change is minimal and isolated to the NFAPI message serialization path, with no impact on performance or other system behavior.
