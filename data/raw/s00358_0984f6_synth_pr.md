## Title: Fix errno corruption in vnf_send_p5_msg error path

### Summary
In `vnf_send_p5_msg()`, when `sctp_sendmsg` fails, the error handling code logs the errno value after calling `NFAPI_TRACE`. Since `NFAPI_TRACE` may perform operations that modify errno, the logged error code could be incorrect, leading to misleading debug information and complicating failure diagnosis in nFAPI VNF P5 interface issues.

This patch saves errno immediately after the `sctp_sendmsg` failure to a local variable, ensuring the correct error code is logged regardless of subsequent function calls.

### Changes
- **nfapi/open-nFAPI/vnf/src/vnf.c**: In the error path of `vnf_send_p5_msg()`, save errno to `saved_errno` immediately after `sctp_sendmsg` returns failure, then use the saved value in the `NFAPI_TRACE` call.

### Testing
- Code review confirms errno is now captured before any potential modification by logging infrastructure.
- This prevents debug logs from showing spurious errno values when SCTP send operations fail in the nFAPI VNF P5 message path.
