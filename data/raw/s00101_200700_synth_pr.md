## Title: Fix incorrect channel estimates pointer in nr_ulsch_mmse_2layers call

### Summary
Correct a parameter error in the uplink demodulation module where the wrong channel estimates pointer was passed to the MMSE receiver for 2-layer MIMO processing. The `inner_rx` function incorrectly used `chFext` instead of `pusch_vars->ul_ch_estimates_ext` when calling `nr_ulsch_mmse_2layers`, causing the equalizer to operate on stale or incorrect channel state information. This fix ensures proper channel compensation for 2-layer PUSCH transmissions.

### Changes
- `openair1/PHY/NR_TRANSPORT/nr_ulsch_demodulation.c`: Line 1368 - Changed the 6th argument in `nr_ulsch_mmse_2layers()` call from `(int **)chFext` to `(int **)pusch_vars->ul_ch_estimates_ext`

### Implementation Details
The bug was located in the 2-layer MIMO branch of the uplink receiver chain. The `chFext` variable contains compensated channel estimates, while `ul_ch_estimates_ext` contains the raw channel estimates needed by the MMSE algorithm for proper noise covariance estimation and interference suppression. The corrected argument order matches the function signature and ensures the MMSE receiver receives the appropriate channel data structure.

### Testing
- Verified successful compilation of `nr-softmodem` target
- Code review confirms the corrected parameter type matches the expected `int32_t **` channel estimates array format used throughout the PHY layer
