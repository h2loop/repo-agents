## Title: Add NULL pointer check in pdcp_advance_rx_window to prevent NGAP-triggered crashes

### Summary
The PDCP sequence manager lacked a NULL pointer guard at the entry of `pdcp_advance_rx_window()`, which could cause segmentation faults when called with an invalid PDCP entity pointer during NGAP interface error scenarios between gNB and AMF. Under certain race conditions or protocol failure paths, the downstream NGAP procedures could invoke this function before the PDCP entity is fully initialized or after it has been freed, leading to undefined behavior and service disruption.

This fix adds an explicit NULL check at the function entry point to fail gracefully with an error log instead of crashing. This prevents the cascading failures observed in NGAP procedures and improves system robustness during transient error conditions.

### Changes
- `openair2/LAYER2/PDCP_v10.1.0/pdcp_sequence_manager.c`: Added NULL pointer validation for `pdcp_entity` parameter in `pdcp_advance_rx_window()` at function entry. Returns `false` and logs error if NULL.

### Implementation Details
The check is placed at the very beginning of the function, before any dereferencing occurs. This follows defensive programming practices and matches the existing error handling pattern used elsewhere in the PDCP module. The error log uses the standard PDCP logging macro for consistency.

### Testing
- Verified the fix with unit tests that simulate NULL pointer scenarios
- Ran NGAP interface stress tests to ensure no regression in normal operation
- Confirmed error logs are properly emitted when the condition is triggered
