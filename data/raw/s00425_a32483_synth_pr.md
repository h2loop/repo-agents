## Title: Fix HARQ retransmission parameter override in UL scheduler

### Summary
The UL scheduler was incorrectly recalculating MCS and RB allocations during HARQ retransmissions. When a HARQ process entered retransmission mode (round_UL > 0), the code path would still call `calculate_max_mcs_min_rb()`, potentially overriding the original transmission parameters with newly calculated values. This caused mismatched redundancy versions and incorrect HARQ behavior, as retransmissions must use the same MCS and resource allocation as the initial transmission.

This fix adds an early `continue` statement to skip parameter recalculation for active HARQ retransmissions. When a HARQ process is in retransmission mode, the scheduler now preserves the original MCS, redundancy version, and RB allocation instead of recalculating them.

### Changes
- `openair2/LAYER2/MAC/pre_processor.c`: Added early continue in `rr_ul_run()` to bypass `calculate_max_mcs_min_rb()` for HARQ retransmissions, ensuring consistent parameters across retransmission rounds.

### Testing
- Verified HARQ retransmissions maintain consistent MCS and redundancy versions across multiple rounds
- Confirmed UL throughput stability under packet loss conditions
- Validated no regression in scheduling behavior for new transmissions
