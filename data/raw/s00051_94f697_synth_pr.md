## Title: Fix incorrect type cast in UL config request packing function

### Summary
Correct an incorrect type cast in `pack_ul_config_request_ulsch_rel8_value()` that caused misaligned memory access when packing UL_SCH PDU data. The `tlv` parameter points to raw data after the TL (tag-length) header, not directly to a `nfapi_ul_config_ulsch_pdu_rel8_t` structure. The previous implementation incorrectly cast the pointer to the struct type, leading to potential undefined behavior and packing errors.

The fix replaces the direct struct pointer cast with explicit offset-based field access, properly accounting for the `nfapi_tl_t` header that precedes the PDU data in memory.

### Changes
- `nfapi/open-nFAPI/nfapi/src/nfapi_p7.c`: 
  - Rewrote `pack_ul_config_request_ulsch_rel8_value()` to access fields using byte offsets from the data pointer
  - Added clarifying comments documenting each field's offset
  - Replaced struct pointer dereferencing with explicit type casts at calculated offsets

### Implementation Details
The function now treats `tlv` as a raw `uint8_t*` data buffer and calculates each field's position manually:
- `handle` at offset 0 (uint32_t)
- `size` at offset 4 (uint16_t) 
- `rnti` at offset 6 (uint16_t)
- Remaining fields at subsequent byte offsets (8-17)

This approach correctly handles the packed data layout and avoids alignment issues present in the original implementation.

### Testing
- Verified correct packing of UL_SCH PDUs in unit tests
- Validated against nfapi message format specifications
- Confirmed no regression in UL configuration message generation
