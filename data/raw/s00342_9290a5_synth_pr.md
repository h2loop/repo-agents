## Title: Fix UL failure timer duration to comply with 3GPP specifications

### Summary
The uplink failure timer in `nr_mac_trigger_ul_failure()` was incorrectly set to 30 seconds with subcarrier spacing scaling, violating 3GPP timing requirements. The specification mandates a 5-second timer that must not be scaled by subcarrier spacing. This fix corrects the timer value to ensure protocol compliance and proper UE release timing.

### Changes
- `openair2/LAYER2/NR_MAC_gNB/gNB_scheduler_primitives.c`: In `nr_mac_trigger_ul_failure()` at line 3167, change `sched_ctrl->ul_failure_timer` from `30000 << subcarrier_spacing` to `5000`. Update the comment to clarify the 5-second duration per 3GPP and explicitly note that the timer should not be scaled by subcarrier spacing.

### Implementation Details
This is a straightforward constant correction. The timer value now directly matches the 3GPP-specified 5-second duration without any bit-shifting operations. The updated comment prevents future regressions by explicitly documenting the requirement.
