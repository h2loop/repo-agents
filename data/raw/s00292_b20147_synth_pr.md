## Title: Fix race condition in PDN disconnect procedure between timer expiry and message processing

### Summary
The PDN disconnect procedure in the NAS UE layer had a race condition between the T3492 retransmission timer expiry handler and the processing of PDN disconnect reject messages. When a reject message arrived while the timer was expiring, both code paths would concurrently access shared resources including the retransmission counter and procedure transaction state. This caused inconsistent state updates, potential memory corruption, and intermittent crashes in the UE NAS layer.

This fix introduces a mutex to serialize access to these shared resources. The timer handler and message processing function now acquire the same lock before modifying any shared state, eliminating the race condition. The mutex is implemented as a static initializer to ensure proper cleanup and avoid dynamic allocation.

### Changes
- `openair3/NAS/UE/ESM/PdnDisconnect.c`: 
  - Added static `pdn_disconnect_mutex` with `PTHREAD_MUTEX_INITIALIZER`
  - Wrapped timer stop, state changes, and bearer context modifications in `esm_proc_pdn_disconnect_reject()` with mutex lock/unlock
  - Wrapped retransmission counter increment and timeout processing in `_pdn_disconnect_t3492_handler()` with mutex lock/unlock
  - Ensured all exit paths properly release the mutex

### Testing
- Verified the fix through code inspection and static analysis to confirm all shared resource access paths are protected
- Ran ThreadSanitizer on the NAS UE component under simulated network conditions; no data race warnings reported in the PDN disconnect procedure
- Validated normal PDN connectivity establishment and release operations continue to work correctly with no functional regression
