## Title: Add proper error logging for SIGTERM/SIGINT shutdown handler

### Summary
The `catch_sigterm()` signal handler in the ITTI (Inter-Task Interface) component currently only writes a plain text message to stdout when SIGTERM or SIGINT is received. This approach bypasses the centralized logging infrastructure, making shutdown events invisible to production logging systems that rely on severity levels, timestamps, and structured formatting for debugging and monitoring. This hinders root cause analysis during unexpected process terminations.

This fix adds a proper error-level log statement using the `LOG_E` macro to ensure shutdown events are captured consistently alongside other critical system events. The existing stdout write is preserved for backward compatibility and immediate console visibility.

### Changes
- `common/utils/ocp_itti/intertask_interface.cpp`: Added `LOG_E(ITTI, "Caught SIGTERM/SIGINT, shutting down\n");` at line 449 in the `catch_sigterm()` function, immediately after the existing stdout write call.

### Testing
- Verified the error-level log message appears correctly in system logs when sending SIGTERM to a running process
- Confirmed no change in shutdown behavior or timing
- Ensured proper integration with the ITTI logging subsystem for downstream log aggregation tools
