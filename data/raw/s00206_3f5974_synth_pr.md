## Title: Fix missing transform precoding IE in PUSCH signaling messages

### Summary
The transform precoding parameter calculated by `get_transformPrecoding()` was not being persisted in the UL_BWP structure, causing a missing mandatory Information Element (IE) in downstream PUSCH signaling messages. The function computed the value for immediate DCI encoding use, but downstream message preparation functions could not access this critical parameter when constructing PUSCH PDUs.

This fix stores the transform precoding value in the UL_BWP structure immediately after calculation, ensuring it is available for all subsequent signaling message construction that depends on this mandatory IE.

### Changes
- `openair2/LAYER2/NR_MAC_COMMON/nr_mac_common.c`: Store transform precoding result in `UL_BWP->transform_precoding` after calling `get_transformPrecoding()` during DCI size calculation
- `openair2/LAYER2/NR_MAC_gNB/gNB_scheduler_RA.c`: Store transform precoding result in `ul_bwp->transform_precoding` during Msg3 retransmission preparation

### Implementation Details
The transform precoding value is now cached in the BWP structure at the point of calculation rather than being passed as a transient parameter. This ensures consistency across the MAC layer and prevents the missing IE issue in PUSCH PDU preparation functions that reference this value later in the call chain. The change maintains the existing API while adding the necessary state persistence.
