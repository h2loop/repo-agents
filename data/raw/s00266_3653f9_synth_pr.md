## Title: Fix buffer overflow in emm_send_authentication_response

### Summary
Fix a buffer overflow vulnerability in the NAS UE authentication response handling. The function `emm_send_authentication_response` was directly assigning an OctetString structure, which copies the pointer rather than the underlying data. This can lead to buffer overflow, memory corruption, and potential security exploits when the authentication response parameter is processed.

### Changes
- `openair3/NAS/UE/EMM/SAP/emm_send.c`: Replace direct OctetString assignment with `DUP_OCTET_STRING` macro to properly duplicate the authentication response parameter data.

### Implementation Details
The vulnerability was located at line 779 in `emm_send_authentication_response()`. The code was using `emm_msg->authenticationresponseparameter.res = *msg->res;` which performs a shallow copy of the OctetString structure. This copies the `buffer` pointer but not the actual data, leading to potential buffer overflow when the receiving code accesses the data. The fix uses the `DUP_OCTET_STRING` macro, which properly allocates memory and copies the data content, preventing the overflow.

### Testing
- This is a security fix that addresses a buffer overflow vulnerability
- The change follows the established pattern used elsewhere in the NAS codebase for handling OctetString duplication
- Recommend testing with authentication scenarios to ensure proper memory handling and no regressions
