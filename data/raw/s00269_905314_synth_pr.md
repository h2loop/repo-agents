## Title: Fix integer overflow in turbo encoder buffer offset calculation

### Summary
Fix integer overflow in `threegpplte_turbo_encoder_sse()` that occurs when processing large input blocks. The function calculates `input_length_bits = input_length_bytes << 3` using an `unsigned short` variable, which overflows for inputs larger than 8191 bytes. This overflow leads to incorrect buffer offset calculations (`input_length_bits * 3`), causing out-of-bounds memory writes during turbo encoding and potential heap corruption.

The fix promotes `input_length_bits` to `unsigned int` and explicitly casts `input_length_bytes` to `unsigned int` before the shift operation. Additionally, the pointer arithmetic for output buffer positioning is hardened by casting to `size_t` to prevent overflow on 64-bit platforms when handling large code blocks.

### Changes
- `openair1/PHY/CODING/3gpplte_sse.c`: 
  - Changed `input_length_bits` variable type from `unsigned short` to `unsigned int`
  - Added explicit cast to `unsigned int` for the left shift operation
  - Cast `input_length_bits` to `size_t` before multiplication in pointer arithmetic

### Implementation Details
The original code used `unsigned short` (16-bit) for `input_length_bits`, which overflows when `input_length_bytes` exceeds 8191 (since 8192 * 8 = 65536 > 65535). The multiplication `input_length_bits * 3` for buffer offset calculation exacerbates this issue. Using `unsigned int` (32-bit) provides sufficient range for all practical LTE turbo block sizes, and the `size_t` cast ensures safe pointer arithmetic on 64-bit architectures.
