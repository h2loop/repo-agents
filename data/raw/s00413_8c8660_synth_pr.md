## Title: Fix type confusion in NR MAC CellGroup configuration processing

### Summary
Correct a type confusion bug in `process_CellGroup()` where `CellGroup->mac_CellGroupConfig` was incorrectly declared using typedef `NR_MAC_CellGroupConfig_t*` instead of the actual struct type `struct NR_MAC_CellGroupConfig*`. This mismatch caused undefined behavior when accessing MAC CellGroup configuration fields, potentially leading to memory corruption or crashes. The fix ensures proper type safety and correct struct layout interpretation without changing functional logic.

### Changes
- **openair2/LAYER2/NR_MAC_gNB/config.c**: Updated pointer declaration in `process_CellGroup()` from `NR_MAC_CellGroupConfig_t *mac_CellGroupConfig` to `struct NR_MAC_CellGroupConfig *mac_CellGroupConfig` (line 107)

### Implementation Details
This is a minimal, targeted fix that only corrects the type declaration. The pointer assignment and all subsequent field accesses now use the correct struct type, eliminating the type confusion while preserving existing memory layout and behavior. No other code changes were necessary as the usage pattern remains identical.
