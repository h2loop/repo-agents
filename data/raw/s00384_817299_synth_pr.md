## Title: Add F1AP-aware force_RRC_IDLE telnet command for UE state management

### Summary
Add a new telnet command to manually force a UE into RRC_IDLE state via the F1AP interface. This addresses F1AP interface handling errors between CU and DU that can cause message loss or corruption during UE state transitions. Previously, the telnet interface lacked a mechanism to directly trigger RRC_IDLE transitions, making it difficult to recover from F1AP synchronization issues or debug state machine problems.

The implementation provides controlled UE state transitions with proper validation and F1AP error handling, ensuring clean state cleanup and preventing downstream message corruption.

### Changes
- `common/utils/telnetsrv/telnetsrv_ciUE.c`: 
  - Added `force_RRC_IDLE()` function to validate UE ID, check current sync state, and trigger RRC_IDLE transition
  - Registered new "force_RRC_IDLE" command in the telnet command table
  - Includes comprehensive error handling and user feedback via telnet print interface

### Implementation Details
- Uses `nr_ue_mac_handle_rrc_idle_transition()` for the actual state transition with integrated F1AP error handling
- Validates UE ID positivity and existence before attempting transition
- Provides clear status messages indicating success or specific failure reasons
- Maintains thread safety by leveraging existing MAC layer state management primitives

### Testing
- Verified command execution for valid UE IDs results in successful RRC_IDLE transition
- Confirmed proper error handling for negative and non-existent UE IDs
- Tested F1AP message flow during forced transitions to ensure no message loss or corruption
- Validated telnet command registration and help text display
