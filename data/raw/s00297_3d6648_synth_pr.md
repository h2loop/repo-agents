## Title: Replace magic numbers with symbolic constants in E1AP bearer context management

### Summary
Replace hardcoded numeric literals with symbolic constants to improve code readability and maintainability in the E1AP bearer context management module. The magic number `1` was used in multiple locations to represent two distinct constraints: the maximum number of DRBs per PDU session and the maximum RAN count. These constraints are now explicitly defined as `E1AP_MAX_DRB_PER_PDU_SESSION` and `E1AP_MAX_RAN_COUNT`, making the code's intent clearer and reducing the risk of errors when these limits need to be modified in the future.

### Changes
- `openair2/E1AP/lib/e1ap_bearer_context_management.c`: Replace four instances of magic number `1` with appropriate symbolic constants in validation checks within `e1_decode_pdu_session_to_setup_item()`, `decode_E1_bearer_context_setup_request()`, `decode_E1_bearer_context_setup_response()`, and `e1_decode_pdu_session_to_setup_mod_item()`.

### Testing
- This is a pure refactoring change with no functional modifications.
- Code behavior remains identical; only the representation of constant values has changed.
- Compilation successful with no new warnings or errors.
