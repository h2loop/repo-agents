## Title: Fix type confusion in paramlist array handling

### Summary
Fix a type confusion bug in the configuration library where `paramlist_def_t.paramarray` was declared as `paramdef_t **` but used as a flat array of `paramdef_t` structures. The incorrect type declaration led to incompatible pointer casting and unnecessary double-pointer indirection. This change corrects the type to `paramdef_t *` and updates the allocation/access logic to use a single contiguous memory block, improving type safety and memory efficiency.

### Changes
- **common/config/config_paramdesc.h**: Changed `paramarray` field type from `paramdef_t **` to `paramdef_t *` in `paramlist_def_t` structure
- **common/config/libconfig/config_libconfig.c**: 
  - Updated allocation to create single block: `ParamList->numelt * numparams * sizeof(paramdef_t)`
  - Replaced double-pointer indexing with flat array arithmetic: `&ParamList->paramarray[i * numparams]`
  - Simplified parameter passing to `config_libconfig_get()` using direct pointer

### Implementation Details
The original code allocated an array of pointers (`paramdef_t **`) then allocated each element separately, but accessed them as a flat 2D array. The fix uses a single allocation for the entire 2D array and calculates element addresses with `i * numparams` offset arithmetic. This eliminates the type mismatch while maintaining the same logical structure and memory layout that the rest of the codebase expects.

### Testing
This is a type safety and memory layout correction that preserves existing behavior. Validation should be performed through existing configuration parsing tests to ensure parameter list reading functionality remains unchanged.
