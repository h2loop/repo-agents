## Title: Fix incorrect type cast in PRS TOA calculation

### Summary
The PRS (Positioning Reference Signal) time-of-arrival (TOA) calculation in `nr_prs_channel_estimation` was performing division on `prs_toa` without proper type casting. Since `prs_toa` is an integer type, the division operation was subject to integer arithmetic rules, causing precision loss before the result was stored in the floating-point field `dl_toa`. This led to inaccurate TOA measurements that could degrade positioning accuracy and cause downstream processing errors.

The fix explicitly casts `prs_toa` to `float` before division, ensuring proper floating-point arithmetic throughout the calculation. This maintains the precision of the TOA estimate and prevents quantization errors in the positioning reference signal processing pipeline.

### Changes
- `openair1/PHY/NR_UE_ESTIMATION/nr_dl_channel_estimation.c`: Added explicit `(float)` cast to `prs_toa` variable in the TOA calculation at line 462 to ensure floating-point division precision.

### Testing
- Verified that PRS TOA values now maintain proper fractional precision
- Confirmed no regression in RSRP/SNR estimation accuracy
- Validated positioning measurements remain stable across multiple subframes
