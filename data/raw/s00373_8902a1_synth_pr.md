## Title: Add bounds checking to prevent out-of-bounds access in MCH SDU processing

### Summary
The `ue_send_mch_sdu()` function in the UE MAC layer performs multicast channel (MCH) SDU parsing without validating buffer boundaries before memory access. This leads to potential out-of-bounds reads when accessing `sdu[0]` and `sdu[1]` for logging, iterating over MCH scheduling information elements, and dereferencing payload pointers. The lack of bounds checking can cause undefined behavior or crashes when processing malformed or truncated SDUs.

This fix adds comprehensive bounds validation throughout the function: SDU length is verified before accessing specific indices, payload pointer arithmetic is checked against buffer limits, and loop counters are capped to prevent array overflow. These defensive checks ensure all memory accesses remain within the allocated SDU buffer while preserving existing functionality for valid inputs.

### Changes
- `openair2/LAYER2/MAC/ue_procedures.c`: Added bounds checking before accessing `sdu[0]` and `sdu[1]` in debug logging statements. Added validation to ensure `payload_ptr + rx_lengths[i]` does not exceed `sdu + sdu_len`. Added loop bound limit (`j < 28`) when parsing MCH scheduling information to prevent overflow of `pmch_lcids` and `pmch_stop_mtch` arrays. Added payload bounds check before accessing `payload_ptr[2*j]` and `payload_ptr[2*j+1]`.

### Implementation Details
The bounds checks are implemented as conditional statements that log errors and break loops when violations are detected, preventing further out-of-bounds access. The maximum value of 28 for the scheduling information loop corresponds to the static array size of `UE_mac_inst[module_idP].pmch_lcids` and `pmch_stop_mtch`, ensuring we never write beyond these stack-allocated buffers.

### Testing
- Verified the function gracefully handles SDU buffers shorter than 2 bytes
- Confirmed MCH scheduling information parsing stops at array bounds
- Ensured no functional regression for valid, well-formed MCH SDUs
