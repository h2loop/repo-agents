## Title: Fix type confusion in PDCP SRB entity creation

### Summary
Fix a type confusion bug in the PDCP layer where a UE context pointer was incorrectly passed as a callback function parameter during SRB (Signaling Radio Bearer) PDCP entity creation. The `deliver_pdu_data` parameter expects a function pointer for PDU delivery notifications, but for SRBs this callback is unused. Passing the `ue` pointer (nr_pdcp_ue_t*) created a type mismatch that could lead to undefined behavior if the code ever attempted to invoke it as a function. The fix sets this parameter to `NULL` with a clarifying comment, matching the intended design for SRB entities.

### Changes
- `openair2/LAYER2/nr_pdcp/nr_pdcp_oai_api.c`: Changed the `deliver_pdu_data` argument from `ue` to `NULL` in the SRB creation path inside `add_srb()`. Added comment explaining that PDU delivery callbacks are not used for SRBs.

### Implementation Details
The type confusion occurred in the `add_srb()` function when calling the PDCP entity creation API. While DRBs (Data Radio Bearers) require PDU delivery callbacks for data path notifications, SRBs use a separate SDU delivery mechanism. The `deliver_pdu_data` parameter should always be `NULL` for SRBs to avoid potential function pointer dereference issues and to clearly indicate the callback is intentionally unused.

### Testing
- Verified clean compilation with `-Werror=cast-function-type`
- Confirmed SRB establishment in basic UE attach scenarios
- Validated no regression in PDCP data path for DRBs
