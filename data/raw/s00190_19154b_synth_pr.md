## Title: Fix uninitialized EMM-SAP primitive memory in NAS UE lower layer

### Summary
The NAS UE lower layer functions were passing uninitialized stack memory to the EMM-SAP interface, causing undefined behavior. The `emm_sap_t` primitives in `lowerlayer_success()`, `lowerlayer_failure()`, and `lowerlayer_release()` were allocated on the stack without zero-initialization before field assignment. This allowed garbage values in padding bytes and unused struct members to be interpreted as flags/bitmasks downstream in `emm_sap_send()`, leading to unpredictable NAS state machine behavior and potential security vulnerabilities.

This patch adds explicit zero-initialization using `memset()` before any field assignments, ensuring deterministic behavior and preventing spurious flag values from propagating through the EMM-SAP interface.

### Changes
- `openair3/NAS/UE/EMM/LowerLayer.c`: Added `memset(&emm_sap, 0, sizeof(emm_sap_t))` at the start of `lowerlayer_success()`, `lowerlayer_failure()`, and `lowerlayer_release()` functions.

### Implementation Details
The fix follows defensive C programming practices for structures containing unions and bitfields. Zero-initialization guarantees that all padding bytes and unused members are cleared before the primitive is populated, eliminating undefined behavior across different compiler optimizations and architectures. This is particularly critical for EMM-SAP primitives that cross module boundaries where downstream code may inspect or serialize the entire struct memory region.
