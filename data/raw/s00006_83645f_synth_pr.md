## Title: Add job list cleanup to prevent stale protocol state in OMG subsystem

### Summary
The OMG (OpenAirInterface Mobility Generator) subsystem was not properly cleaning up job lists after procedure completion or failure, leading to stale protocol state and memory leaks. The `nodes_avgspeed()` function in `job.c` accessed job list entries without proper validation, risking division by zero and NULL pointer dereferences when the list contained invalid entries.

This patch introduces a `clear_job_list()` function that safely traverses and frees all memory associated with job lists, including nested mobility structures. The function is exposed via `defs.h` for use by OMG callers to ensure proper cleanup after mobility procedures complete.

### Changes
- `openair2/UTIL/OMG/defs.h`: Added declaration for `clear_job_list()` with documentation
- `openair2/UTIL/OMG/job.c`: Implemented `clear_job_list()` with defensive NULL checking at each nesting level. The function frees mobility structures, node pairs, and job list nodes in the correct order, then nullifies the list pointer.

### Testing
- Verified clean compilation with GCC 11
- Confirmed no memory leaks in job list lifecycle via Valgrind on mobility simulation scenarios
- Tested with empty job lists, single-entry lists, and multi-entry lists to ensure robust cleanup
