## Title: Add null check for PUSCH config before computing precoding information

### Summary
The `config_uldci()` function in the NR MAC gNB scheduler was calling `compute_precoding_information()` without validating that the `pusch_Config` pointer is non-NULL. This could lead to null pointer dereference and crashes when PUSCH configuration is not available during DCI preparation. This patch adds a defensive null check to ensure precoding information is only computed when the configuration is valid, preventing potential segmentation faults and improving scheduler robustness.

### Changes
- `openair2/LAYER2/NR_MAC_gNB/gNB_scheduler_primitives.c`: Added null pointer validation for `pusch_Config` before invoking `compute_precoding_information()` in the `config_uldci()` function. The precoding computation is now conditionally executed only when `pusch_Config` is valid, with an explicit comment explaining the guard.

### Testing
- Static analysis confirms the null check eliminates the potential null dereference path
- Code review validates the fix maintains correct behavior when configuration is present
- The change is minimal and defensive, preserving existing functionality while adding safety
