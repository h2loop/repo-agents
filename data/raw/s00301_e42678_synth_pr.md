## Title: Fix type confusion in NAS timer callback arguments

### Summary
The NAS timer implementation suffered from a type confusion vulnerability where raw callback arguments (`void *args`) were passed directly to the ITTI timer system. Downstream code incorrectly cast this pointer to an incompatible struct type, risking memory corruption, crashes, and undefined behavior when timers fired.

The fix introduces a `nas_timer_wrapper_t` structure that properly encapsulates both the original NAS timer callback function pointer and its arguments. This wrapper ensures type safety by providing a well-defined interface between the NAS timer layer and the ITTI timer subsystem.

### Changes
- **openair3/NAS/COMMON/UTIL/nas_timer.c**:
  - Added `nas_timer_wrapper_t` structure containing `nas_timer_callback_t cb` and `void *args` fields
  - Modified `nas_timer_start()` to allocate wrapper, store original callback/args, and pass wrapper to `timer_setup()`
  - Updated timer expiry handler to unwrap and invoke the original callback with correct arguments

### Implementation Details
The wrapper pattern preserves the original callback semantics while preventing incompatible pointer casts. The ITTI timer system now receives a properly typed `nas_timer_wrapper_t*` instead of an opaque `void*` that could be misinterpreted downstream. Memory management follows the existing pattern where the wrapper is freed after timer expiration or cancellation.

### Testing
- Verified timer execution for NAS registration and periodic update procedures
- Confirmed no memory leaks via valgrind on timer create/cancel cycles
- Validated that callback arguments are correctly passed through the wrapper layer without corruption
