## Title: Fix F1AP message corruption via deep copy in CU-UP bearer context handling

### Summary
The F1AP interface between CU and DU was experiencing message loss and corruption due to improper memory copying in bearer context setup and modification functions. The root cause was the use of `memcpy()` to copy ITTI message structures containing nested arrays and potential pointer fields, which could lead to shallow copies and memory corruption when messages are queued or retransmitted.

This fix replaces the unsafe `memcpy()` operations with explicit deep copy logic that copies each field individually. Primitive fields are assigned directly, fixed-size arrays (encryption/integrity keys) use `memcpy()` with explicit size, and variable-length PDU session arrays are copied element-by-element in loops. This ensures proper isolation between the source request and the ITTI message payload.

### Changes
- `openair2/RRC/NR/cucp_cuup_e1ap.c`: Replaced shallow `memcpy()` in `cucp_cuup_bearer_context_setup_e1ap()` and `cucp_cuup_bearer_context_mod_e1ap()` with field-by-field deep copy logic. Added explicit copying for all primitive fields, encryption/integrity keys, and PDU session arrays.

### Implementation Details
The deep copy approach ensures that ITTI messages are self-contained and safe for asynchronous processing. Each field is copied individually to prevent any pointer aliasing or layout-dependent issues. The PDU session arrays are bounded by `numPDUSessions` and `numPDUSessionsMod` counters to prevent buffer overruns.
