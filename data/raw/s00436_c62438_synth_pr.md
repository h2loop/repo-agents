## Title: Add defensive clamping to prevent buffer overruns in OTG TARMA model

### Summary
The TARMA (Time-AutoRegressive Moving Average) traffic model in the OTG subsystem can generate extreme values that lead to buffer overruns and excessive memory allocation. When `tarmaCalculateSample()` returns very large positive or negative values, these are used directly for memory allocation and string generation, causing potential buffer overruns in downstream string operations. This fix implements multi-layer defensive clamping to ensure all size and timing values remain within safe, bounded ranges.

### Changes
- `openair2/UTIL/OTG/otg_models.c`: Clamp `tarmaCalculateSample()` output to ±1,000,000 to prevent extreme values from propagating through the system
- `openair2/UTIL/OTG/otg_tx.c`: 
  - Clamp inter-departure time (`idt`) to [0, 1,000,000] in `time_dist()` TARMA case
  - Clamp packet size to [PAYLOAD_MIN, PAYLOAD_MAX] in `size_dist()` TARMA case
  - Add size validation in `random_string()` to prevent excessive memory allocation and ensure safe string operations

### Implementation Details
The clamping thresholds are chosen to be large enough for realistic traffic modeling while preventing integer overflow and buffer overrun scenarios. Warning logs are added when clamping occurs to help identify potential model misconfiguration. The fix follows defense-in-depth principles by validating at multiple points in the data flow, ensuring that even if one clamp is bypassed, subsequent layers will catch out-of-bounds values.
