## Title: Fix incorrect LTE subframe count causing power control errors

### Summary
The `subframe_num()` function in `phy_procedures_lte_common.c` incorrectly returned 6-9 subframes for TDD configurations instead of the standard 10 subframes per frame. This miscalculation propagated downstream, causing power control algorithms to compute incorrect transmit power levels. According to 3GPP LTE specifications, all frames (both FDD and TDD) contain exactly 10 subframes; TDD configuration only affects the uplink/downlink allocation pattern, not the total subframe count. This patch corrects the fundamental frame structure error by returning the proper constant value.

### Changes
- `openair1/SCHED/phy_procedures_lte_common.c`: Simplified `subframe_num()` to always return 10 subframes
  - Removed incorrect TDD-specific branching logic that returned 6-9 subframes based on `tdd_config`
  - Eliminated now-obsolete error handling for "unsupported TDD configurations"
  - Added clarifying comment that TDD config affects UL/DL pattern, not subframe count

### Impact
This fix resolves power control miscalculations that depended on accurate subframe counting, particularly affecting TDD deployments. The change is minimal and aligns the implementation with 3GPP LTE standards (TS 36.211).

### Testing
- Verified against 3GPP TS 36.211 frame structure specifications
- Validated that downstream power control functions now receive correct subframe count
- Confirmed no regression in FDD mode (still returns 10 as before)
