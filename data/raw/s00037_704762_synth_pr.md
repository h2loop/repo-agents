## Title: Fix stale protocol state cleanup in MCH generation and FEC encoding

### Summary
The LTE PHY MCH (Multicast Channel) generation path fails to reset protocol state after procedure completion, causing stale HARQ process state and lingering FIFO elements to persist. After `generate_mch_khz_1dot25()` completes MCH generation, the HARQ process remains active with non-zero round count and status other than idle. Similarly, the FEC encoding function `dlsch_encoding_fembms_pmch()` can leave residual elements in the response FIFO after processing. This stale state can corrupt subsequent MCH transmissions and cause resource leaks.

This patch ensures proper cleanup by resetting the MCH HARQ process to idle state after each generation and draining any remaining elements from the FEC encoding FIFO upon completion.

### Changes
- `openair1/PHY/LTE_TRANSPORT/pmch.c`: Reset HARQ process state after MCH generation by setting `harq_processes[0]->round = 0` and `status = SCH_IDLE`
- `openair1/PHY/LTE_TRANSPORT/dlsch_coding.c`: Add cleanup loop to poll and delete any remaining elements in `respEncode` FIFO after encoding completes

### Implementation Details
The HARQ cleanup explicitly targets the MCH process (index 0) to reset transmission round count and return status to `SCH_IDLE`. The FIFO cleanup uses `pollNotifiedFIFO()` in a while loop to ensure all residual elements are removed, preventing accumulation of stale protocol state between MCH transmissions.
