## Title: Fix memory leaks in GTP-U data path

### Summary
The GTP-U interface has memory management issues where allocated buffers are not properly freed after use, leading to memory leaks and potential double-free errors. Specifically, the `extensionHeader` structure in `gtpv1uSendDlDeliveryStatus()` and the data buffer in the `GTPV1U_TUNNEL_DATA_REQ` message processing path are not released after being passed to downstream functions.

This fix adds the missing `free()` and `itti_free()` calls to properly release these buffers after they have been processed. No functional behavior is changed.

### Changes
- `openair3/ocp-gtpu/gtp_itf.cpp`: 
  - Added `free(extensionHeader)` after `gtpv1uCreateAndSendMsg()` in `gtpv1uSendDlDeliveryStatus()` to release the extension header buffer
  - Added `itti_free(TASK_GTPV1_U, GTPV1U_TUNNEL_DATA_REQ(message_p).buffer)` after `gtpv1uSend()` in the `GTPV1U_TUNNEL_DATA_REQ` case to properly release the ITTI-allocated message buffer

### Testing
- Ran Valgrind on the GTP-U data path to verify no memory leaks
- Executed the `nr-sa-attach-detach` CI pipeline to confirm no functional regressions
