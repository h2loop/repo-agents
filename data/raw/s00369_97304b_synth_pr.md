## Title: Clear stale protocol state in NR DL-SCH layer demapping

### Summary
The NR downlink shared channel (DL-SCH) demodulation module had a bug where LLR (Log-Likelihood Ratio) buffers retained stale data from previous procedure calls. This occurred because the `layer_llr` pointer array was not initialized to NULL, and the `nr_dlsch_layer_demapping` function did not clear buffers when codewords were absent (marked by `codeword_TB0/1 == -1`). This stale state could cause incorrect demodulation results in subsequent operations, particularly after procedure failures or when transitioning between different transmission configurations.

This fix ensures proper state management by initializing all layer LLR pointers to NULL and explicitly clearing buffer contents when a codeword is inactive, preventing data leakage across procedure calls.

### Changes
- `openair1/PHY/NR_UE_TRANSPORT/nr_dlsch_demodulation.c`:
  - Initialize `layer_llr` array pointers to NULL in `nr_rx_pdsch()` to prevent uninitialized access
  - Add stale state clearing logic in `nr_dlsch_layer_demapping()` using `memset()` to zero LLR buffers when `codeword_TB1 == -1` or `codeword_TB0 == -1`

### Testing
- Verified that LLR buffers are properly cleared between procedure calls
- Confirmed no regression in normal PDSCH reception flows
- Ensured build completes successfully for NR UE targets
