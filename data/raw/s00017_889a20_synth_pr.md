## Title: Add missing periodicityAndOffset for CSI-IM resources per 3GPP spec

### Summary
The CSI-IM resource configuration function `config_csiim()` was missing the `periodicityAndOffset` field, which is conditionally mandatory per 3GPP TS 38.331. For periodic CSI-IM resources, this field must be present to define the measurement timing. The absence of this field could cause RRC configuration failures or UE compatibility issues when the gNB configures periodic CSI-IM resources for interference measurement alongside CSI-RS.

This fix adds the `periodicityAndOffset` field to CSI-IM resources, copying the periodicity configuration from the associated NZP CSI-RS resource. This ensures proper RRC message construction and 3GPP compliance for periodic CSI-IM resource configuration.

### Changes
- `openair2/LAYER2/NR_MAC_gNB/nr_radio_config.c`: Added allocation and configuration of `periodicityAndOffset` in CSI-IM resources. The field is populated with the same periodicity and offset as the associated NZP CSI-RS resource, ensuring synchronized measurement timing.

### Implementation Details
- Per 3GPP TS 38.331, `periodicityAndOffset` is mandatory for periodic/semi-persistent CSI-IM and absent for aperiodic
- The implementation copies both the `present` enum and the selected periodicity choice from `nzpcsi->periodicityAndOffset`
- This maintains alignment between CSI-RS and CSI-IM resource timing, which is critical for accurate interference measurement
- Added code comments referencing the 3GPP specification for future maintainability
