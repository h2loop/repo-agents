## Title: Fix socket resource leak in pnf_connect error paths

### Summary
The `pnf_connect()` function in the PNF module creates an SCTP socket for nFAPI communication but fails to close it on several error paths, causing socket descriptor leaks. When socket configuration or connection operations fail after socket creation, the function returns error codes without releasing the socket resource. This patch adds proper cleanup by closing the socket descriptor before returning in all error scenarios.

### Changes
- `nfapi/open-nFAPI/pnf/src/pnf.c`: Added missing `close(pnf->p5_sock)` calls in the `pnf_connect()` function before returning error codes in five error paths:
  - After `setsockopt()` failures for `SCTP_INITMSG`, `SCTP_NODELAY`, and `SCTP_EVENTS` options
  - After `bind()` failure when `errno == EINVAL`
  - After `connect()` failure when termination is requested
  - After `getsockopt()` failure for `SCTP_STATUS`

### Implementation Details
The socket descriptor is allocated at the start of `pnf_connect()` via `socket(p->ai_family, SOCK_STREAM, IPPROTO_SCTP)`. Each error path now ensures proper resource cleanup by closing the descriptor before freeing address info structures and returning. This prevents file descriptor exhaustion during repeated connection failures.
