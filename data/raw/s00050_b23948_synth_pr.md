## Title: Fix missing error propagation in ENDC DC prep timeout handler

### Summary
The `rrc_eNB_process_ENDC_DC_prep_timeout` function silently ignored error conditions when processing ENDC DC preparation timeouts, making debugging difficult. Failures such as unknown UE context or invalid RRC state were only logged locally but not propagated to the caller, masking potential issues in dual connectivity setup procedures.

This patch adds proper error propagation by changing the function signature from `void` to `int`, enabling callers to detect and handle failures appropriately.

### Changes
- `openair2/RRC/LTE/rrc_eNB.c`: 
  - Modified `rrc_eNB_process_ENDC_DC_prep_timeout()` to return `int` instead of `void`
  - Added return `-1` for error cases (unknown UE rnti, invalid RRC state)
  - Added return `0` for success case
  - Updated call site in ITTI message processor to check return value and log error on failure

### Implementation Details
The function now returns `-1` when UE context lookup fails or when the UE is not in `RRC_NR_NSA` state. The calling code checks the return value and emits a descriptive error log, improving observability of ENDC DC setup failures without changing the core timeout handling logic.
