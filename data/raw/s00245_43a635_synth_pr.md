## Title: Add NULL state validation in telnet measurement commands

### Summary
The telnet measurement commands `measurcmd_display_phycpu` and `measurcmd_display_phyta` directly dereference the global UE state pointer `PHY_vars_UE_g[0][0]` without validating that the underlying data structures are initialized. When these telnet commands are invoked before the PHY layer completes initialization, this causes segmentation faults and crashes.

The fix adds comprehensive NULL checks at all three levels of the global state hierarchy: the top-level global pointer, the module array, and the UE instance pointer. If any validation fails, the functions now print a clear error message via the telnet interface and return early, preventing crashes and providing actionable feedback to the operator.

### Changes
- `common/utils/telnetsrv/telnetsrv_5Gue_measurements.c`: Added NULL validation in `measurcmd_display_phycpu()` and `measurcmd_display_phyta()` functions. Each function now checks `PHY_vars_UE_g`, `PHY_vars_UE_g[0]`, and the final UE pointer before dereferencing, printing appropriate error messages via the `prnt` callback and returning early on failure.

### Testing
- Verified that telnet commands gracefully handle uninitialized state with clear error messages instead of crashing
- Confirmed no regression in normal operation when UE state is properly initialized
- Manually tested telnet interface during gNB startup sequence to ensure robustness
