## Title: Fix missed scheduling opportunity by adding RSSI.indication callback

### Summary
The nFAPI PNF component was missing a callback mechanism for RSSI.indication messages, causing UEs with pending data to not be scheduled downstream of the `unpack_rssi_indication` function. When an RSSI.indication arrived, there was no hook for the client to trigger immediate UE scheduling, resulting in missed scheduling opportunities and potential latency issues.

This change adds a complete callback chain for RSSI.indication messages, allowing the client to schedule UEs with pending data immediately upon receiving the indication. The implementation follows the existing pattern used for other nFAPI messages (e.g., RSSI.request) and ensures the client can respond to RSSI measurements in a timely manner.

### Changes
- **nfapi/open-nFAPI/pnf/public_inc/nfapi_pnf_interface.h**: Added `rssi_ind` callback function pointer to `nfapi_pnf_config_t` struct with documentation explaining its purpose for UE scheduling
- **nfapi/open-nFAPI/pnf/src/pnf.c**: Implemented `pnf_handle_rssi_indication()` function to unpack and dispatch RSSI.indication messages to the registered callback when PNF and PHY are in RUNNING state
- **nfapi/oai_integration/nfapi_pnf.c**: Registered the client's `rssi_indication` handler by setting `config->rssi_ind = &rssi_indication` during PNF configuration

### Implementation Details
The new handler follows the established nFAPI message flow: unpack the message, validate PNF/PHY state, find the appropriate PHY configuration, and invoke the client callback if registered. This maintains consistency with existing message handlers while closing the gap in the RSSI indication path.

### Testing
- Verified that RSSI.indication messages are now properly dispatched to the client callback
- Confirmed that UE scheduling occurs immediately after RSSI.indication processing
- Ensured no regression in existing nFAPI message handling (RSSI.request, etc.)
- Validated that the PNF correctly ignores indications when not in RUNNING state
