## Title: Add NULL check in nas_timer_restart to prevent segmentation fault

### Summary
The NAS timer management module lacks a NULL pointer check in `nas_timer_restart()` after removing a timer entry from the queue. When `_nas_timer_db_remove_entry()` fails and returns NULL (due to race conditions or corrupted timer state), the code immediately dereferences the pointer to access `te->tv`, causing a segmentation fault. This crash scenario can occur when attempting to restart an active timer that has been invalidated or removed by another thread.

The fix adds defensive validation to check the return value before member access. If the entry pointer is NULL, the function returns `NAS_TIMER_INACTIVE_ID` to signal the failure condition, allowing callers to handle the error gracefully instead of crashing.

### Changes
- `openair3/NAS/COMMON/UTIL/nas_timer.c`: Added NULL check after `_nas_timer_db_remove_entry()` call in `nas_timer_restart()`. If removal fails, the function now returns `NAS_TIMER_INACTIVE_ID` instead of dereferencing an invalid pointer.

### Implementation Details
The validation follows the existing error handling pattern in the NAS timer module, using `NAS_TIMER_INACTIVE_ID` as the standard return code for inactive or invalid timers. The check is positioned immediately after the removal operation and before any structure member access to ensure robustness against concurrent modifications to the timer database.
