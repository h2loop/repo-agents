## Title: Fix incorrect frequency domain assignment bit width in TC-RNTI DCI extraction

### Summary
The `extract_00_tc_rnti` function was using a static bit width value (`frequency_domain_assignment.nbits`) when parsing the frequency domain assignment field from DCI 0_0 messages with TC-RNTI or MSGB-RNTI. This caused incorrect resource block allocation that violated scheduling constraints downstream, as the bit width should vary dynamically based on the actual number of resource blocks (N_RB) configured for the cell.

This patch corrects the bit width calculation by implementing the proper 3GPP formula: `ceil(log2((N_RB * (N_RB + 1)) >> 1))`, which accounts for the combinatorial nature of resource allocation. The function now receives N_RB as a parameter, and the call site is updated to retrieve the correct N_RB value and handle error cases.

### Changes
- `openair2/LAYER2/NR_MAC_UE/nr_ue_procedures.c`: 
  - Modified `extract_00_tc_rnti()` signature to accept `const int N_RB` parameter
  - Updated frequency domain assignment extraction to use dynamic bit width calculation
  - Updated call site in `nr_extract_dci_00_10()` to compute N_RB via `get_nrb_for_dci()` and pass it to `extract_00_tc_rnti()`
  - Added safety check to return `NR_DCI_NONE` if N_RB is zero

### Implementation Details
The fix ensures compliance with 3GPP TS 38.212 by calculating the exact number of bits needed for the frequency domain assignment field based on the bandwidth configuration. The `get_nrb_for_dci()` helper function determines the appropriate N_RB value considering the DCI format and search space type, ensuring correct parsing across different channel bandwidths and configurations.
