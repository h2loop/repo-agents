## Title: Replace magic number with INSTANCE_DEFAULT constant in E1AP

### Summary
The E1AP subsystem was using the literal value `0` as a magic number when calling `itti_alloc_new_message()`, representing a default/unused instance parameter. This practice makes the code's intent unclear and maintenance error-prone. This change replaces all occurrences of the magic number `0` with the symbolic constant `INSTANCE_DEFAULT`, which is already defined in the ITTI interface header.

### Changes
- `openair2/E1AP/e1ap.c`: Added `#include "common/utils/ocp_itti/intertask_interface.h"` to access the `INSTANCE_DEFAULT` constant. Replaced magic number `0` with `INSTANCE_DEFAULT` in five `itti_alloc_new_message()` calls across the file, including in `e1ap_itti_send_sctp_close_association()`, `e1apCUCP_handle_SETUP_REQUEST()`, `e1apCUCP_handle_BEARER_CONTEXT_SETUP_RESPONSE()`, `e1_task_send_sctp_association_req()`, and `cucp_task_send_sctp_init_req()`.

### Implementation Details
The `INSTANCE_DEFAULT` constant is defined in `common/utils/ocp_itti/intertask_interface.h` and has a value of `0`. Using this symbolic constant makes the code self-documenting and aligns with the existing ITTI interface conventions. No functional changes are introduced; this is purely a code clarity improvement.

### Testing
- Verified that `INSTANCE_DEFAULT` is properly defined in the ITTI interface header
- Confirmed all modified functions compile successfully after the change
- Ensured the semantic meaning remains unchanged (the value is still 0, but now self-documenting)
