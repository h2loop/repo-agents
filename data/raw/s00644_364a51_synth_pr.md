## Title: Fix unreachable code paths and improve error handling in detach procedure

### Summary
The EPS bearer context deactivation logic in the UE NAS stack contained unreachable code due to improperly structured conditional statements. The `is_local` code path had a premature return that prevented the network-initiated deactivation path from ever executing, even when intended. Additionally, the detach accept procedure failed to check return codes from critical operations, potentially masking timer and ESM communication failures.

This fix restructures the conditional logic in `esm_proc_eps_bearer_context_deactivate()` to properly separate local and network-initiated deactivation paths, eliminating the unreachable code. In `emm_proc_detach_accept()`, we now properly check return values from `nas_timer_stop()` and `esm_sap_send()`, preserving and propagating errors to ensure robust failure handling during UE-initiated detach completion.

### Changes
- `openair3/NAS/UE/EMM/Detach.c`: Added return code checking for `nas_timer_stop()` and `esm_sap_send()` with proper error propagation. Initialized `rc` to `RETURNok` and preserve the first error encountered.
- `openair3/NAS/UE/ESM/EpsBearerContextDeactivation.c`: Restructured conditional logic to use explicit `if (is_local) { ... } else { ... }` blocks, removing unreachable code in the network-initiated deactivation path.

### Implementation Details
The previous implementation used nested conditionals that created unreachable paths when handling the `ESM_SAP_ALL_EBI` case. The new structure ensures both local deactivation (during detach) and network-initiated deactivation (via NAS messages) are properly handled in distinct code paths. Error handling now follows a "fail-fast and preserve first error" pattern to maintain diagnostic accuracy.
