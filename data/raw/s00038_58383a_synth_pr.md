## Title: Fix PUSCH power control by removing hardcoded delta_pusch override

### Summary
The UE's PUSCH power control was incorrectly forced to zero due to a workaround that overrode the calculated power adjustment value. This caused incorrect transmit power levels regardless of TPC commands in DCI. The workaround (`delta_pusch = 0`) was originally added because the PHY layer was not applying PUSCH TX power correctly. With the PHY issue now resolved, this override prevents proper closed-loop power control from functioning.

This fix removes the hardcoded override, restoring proper power control behavior where the MAC layer calculates the power adjustment based on TPC commands and passes it to the PHY.

### Changes
- `openair2/LAYER2/NR_MAC_UE/nr_ue_scheduler.c`: Removed the line `delta_pusch = 0;` that was overriding the calculated power adjustment value in `nr_config_pusch_pdu()`.

### Implementation Details
The removed line was unconditionally setting `delta_pusch` to zero after the correct value had been calculated from DCI's TPC field and the configured power control parameters. This bypassed the entire power control calculation logic and resulted in static PUSCH transmit power. The fix allows the calculated `delta_pusch` value to be passed to the PHY through the FAPI UL config request, enabling proper closed-loop power control.

### Testing
- Verified that TPC commands in DCI 0_0 and 0_1 now correctly adjust UE PUSCH transmit power
- Confirmed power control operates according to TS 38.213 Table 7.1.1-1 for accumulated and absolute power control modes
- No regression observed in basic attach and data path functionality
