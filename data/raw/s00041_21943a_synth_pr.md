## Title: Initialize UE template RB allocation fields to prevent scheduling violations

### Summary
The `mac_init_cell_params()` function in the MAC layer was not initializing critical resource block (RB) allocation fields in the `UE_TEMPLATE` structure. This left these fields with undefined values that were subsequently used by the uplink scheduler, causing incorrect RB allocation decisions and violating scheduling constraints downstream. The uninitialized fields could lead to out-of-bounds RB table access, incorrect MCS assignments, and invalid pre-allocation state that persisted across UE sessions.

The fix ensures all RB allocation state is properly zeroed during cell parameter initialization, establishing a clean baseline for each UE session and preventing undefined behavior in the scheduler algorithms.

### Changes
- `openair2/LAYER2/MAC/main.c`: 
  - Added initialization for `pre_allocated_nb_rb_ul`, `pre_first_nb_rb_ul`, `pre_assigned_mcs_ul`, `total_allocated_rbs`, `pre_allocated_rb_table_index_ul`, and `assigned_mcs_ul` fields
  - Added nested loop to initialize `nb_rb_ul[]` and `first_rb_ul[]` arrays for all 8 HARQ processes
  - Added local variable `k` for array iteration

### Implementation Details
The initialization sets `pre_allocated_rb_table_index_ul` to -1 to indicate no pre-allocation, and zeros all RB counters and MCS assignments. The 8-element arrays for HARQ process-specific RB allocations are explicitly cleared. This ensures deterministic behavior and prevents stale values from influencing scheduling decisions.
