## Title: Add bounds check for dl_BandwidthP parameter in config_mib

### Summary
The `config_mib` function in the MAC layer uses the `dl_BandwidthP` parameter as an array index in downstream operations without validating its range. Invalid values outside the expected range of 0-5 can cause out-of-bounds memory access, leading to undefined behavior, potential crashes, or silent data corruption. I add an `AssertFatal` check at the function entry to validate the parameter bounds, ensuring early failure with a clear diagnostic message when invalid values are passed.

This defensive programming measure prevents memory safety issues and simplifies debugging by catching configuration errors immediately at the source rather than manifesting as mysterious symptoms later in execution.

### Changes
- `openair2/LAYER2/MAC/config.c`: Add `AssertFatal(dl_BandwidthP >= 0 && dl_BandwidthP < 6, ...)` validation at the start of `config_mib()` to enforce valid parameter range.

### Testing
- Static code analysis confirms `dl_BandwidthP` is used as an array index in subsequent code paths
- Assertion will trigger during development and CI if invalid values are passed
- No functional impact on valid configurations; change is purely protective
